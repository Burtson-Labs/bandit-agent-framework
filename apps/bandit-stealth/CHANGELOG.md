# Changelog

All notable changes to **Bandit Stealth** are listed here. Versions follow the extension's own release cadence.

## 1.7.376

- **Per-model tool-loop caps.** Each model now has a sensible default for how many tool-use rounds it runs before stopping — thorough models like Bandit Logic 2 get more headroom so they don't cut a deep task short, while small local models stay tight. Bandit cloud models take their cap from the gateway (so it can be tuned without an update); local Ollama models use a built-in default. An explicit `BANDIT_MAX_ITERATIONS` (CLI) or `banditStealth.toolUse.maxIterations` (extension) still wins.

## 1.7.375

- **Bandit Logic 2 in the model picker.** The Bandit model picker now pulls the live model list from the gateway, so new models — like **Bandit Logic 2**, a Kimi K2 variant — appear automatically with availability and a short description, no update required. Agentic models also allow more tool-use rounds by default, so they don't cut a deep task short on large repos.

## 1.7.374

- **Clearer Ollama Cloud errors.** When a cloud model returns a subscription/paywall error (some models, like Kimi K2, require a paid Ollama plan), Bandit now points you at the upgrade page and a free local fallback — instead of a misleading "run `ollama signin`" hint. The hint also recognizes both cloud tag shapes (`-cloud` and `:cloud`).

## 1.7.373

- **Ollama Cloud — run frontier models with no local GPU.** Sign in with `ollama signin`, pull a cloud-tagged model, and Bandit uses it like any local one. Ships tuned profiles for the **Kimi K2 family** (native tool calling, 256K context), including the multimodal `kimi-k2.5` / `k2.6` / `k2.7-code` variants. Both cloud tag shapes (`:1t-cloud` and `:cloud`) resolve automatically, and a 401/403 now points you at `ollama signin` instead of a bare error. Apple-silicon **MLX builds** (`qwen3.6:27b-mlx`, `gemma4:26b-mlx`) resolve to their base profiles too. New **Ollama Cloud** section and a recommended-models table in the README.

## 1.7.372

- **Local models on OpenAI-compatible servers can now use native tool calling.** Point Bandit at an OpenAI-compatible endpoint (LM Studio, vLLM, llama.cpp server, OpenRouter, and the like) and tool-capable models route tools through native function-calling instead of the text fallback — more reliable agent turns. Community Ollama models outside the built-in list now get their real context window and tool support auto-detected, and vendor-prefixed model ids (e.g. `Qwen/Qwen2.5-Coder-32B`) resolve to the right tuned profile.
- **Small local models are far steadier.** Leaner per-tier system prompts and a compact tool-description format keep the default model from overflowing its context on the first turn. New recovery checks stop a model from stalling after it "thinks" without acting, claiming completion on a read-only question, fabricating tool results, or wrapping its whole answer in stray code fences.
- **Reasoning streams smoothly in the panel.** Chain-of-thought renders in place without flashing — collapsed by default, expandable while the agent works, and it stays put through tool calls instead of vanishing and reappearing. Markdown the agent includes in its reply renders inline rather than as a raw code block.
- **Writing `.tsx` / JSX files works.** Pre-write validation no longer rejects valid JSX as a syntax error.

## 1.7.371

- **Workbench gains the full IDE shell — Explorer, Source Control, Extensions + a marketplace.** Building on the 1.7.370 redesign, `apps/agent-ui-workbench` now mounts the rest of the VS Code activity-bar surface around the Bandit panel: a file-tree **Explorer**, a **Source Control** sidebar (with a pending-change badge on the activity bar), and an **Extensions** sidebar backed by a browsable **Extensions Marketplace** panel. Adds a **Tool Palette** for inspecting the agent's tool set, an **Ask-User card** that renders the agent's clarifying-question prompts inline, and a **Settings overlay**. The conversation mocks and the workbench markdown renderer were extended to exercise the new surfaces. Still a component-review / screenshot harness — `pnpm -C apps/agent-ui-workbench dev` — not bundled in the extension VSIX.

## 1.7.370

- **Memory: topic memories move to `.bandit/memory/`, and duplicate instruction files consolidate.** Bandit's auto-memory index + topic files (`MEMORY.md` + `memory/`) now live under **`.bandit/memory/`** instead of cluttering the repo root — with back-compat reads from the old location and a one-time migration, so existing projects keep working. `BANDIT.md` stays at the root as the cross-tool entry point (sibling of `CLAUDE.md` / `AGENTS.md`). And when a repo has more than one of `BANDIT.md` / `CLAUDE.md` / `AGENTS.md`, Bandit now **deduplicates overlapping content at load** (no more double-counting the same instructions into context) and can **consolidate** them into one source of truth — a symlink on macOS/Linux, an exact copy with a drift warning on Windows.

## 1.7.369

- **Built-in pre-tool security guard (opt-in).** A new safety net catches the handful of tool calls that are almost never legitimate — `rm -rf /`, a remote script piped into a shell (`curl … | sh`), raw disk writes (`dd of=/dev/sda`, `mkfs`), fork bombs, recursive `chmod`/`chown` on `/`, writes or redirects into system/credential paths (`/etc`, `~/.ssh`, `~/.aws`), and reading a credential file then shipping it over the network in one command — and blocks them before they run. Off by default; enable with `"security": { "guard": { "enabled": true } }` in `~/.bandit/settings.json` (global) or a workspace `.bandit/settings.json`, with optional `blockCommands` / `protectPaths` to extend it. It's defense-in-depth against the model footgunning — runs *before* your own hooks and the approval gate, and behaves identically in the IDE and the CLI. (14 new contract tests.)
- **Hooks, permissions, and the guard now apply globally.** `~/.bandit/settings.json` is read and merged *under* the workspace `.bandit/settings.json`, so a `PreToolUse` / `PostToolUse` / `Stop` hook, a permission rule, or the security guard you configure once now applies to every repo — with per-project files still layering overrides on top. Hooks are general-purpose, not just guardrails: only `PreToolUse` blocks (non-zero exit aborts the call); `PostToolUse` / `Stop` are fire-and-forget for formatters, version bumps, notifications, CI triggers, and the like, with `{{name}}` / `{{primary}}` / `{{duration}}` placeholders.
- **Opt-in usage telemetry (OpenTelemetry / OTLP).** Bandit can now emit one trace per turn plus usage metrics — tokens, time-to-first-token, turn duration — over standard OTLP/HTTP. **Off by default; nothing leaves your machine unless you turn it on.** Point it at any collector via `"telemetry"` in `~/.bandit/config.json` (or `BANDIT_TELEMETRY=1` + `BANDIT_OTLP_ENDPOINT`); Bandit Cloud users get a zero-config default. Prompt and response text are **never** sent — only metadata, run through secret redaction first — and `mode: "metrics-only"` drops traces entirely.

## 1.7.365

- Reasoning panels no longer flash mid-word during streaming. When a fenced reasoning block was arriving chunk-by-chunk, the markdown renderer would briefly show a "reasoning (1 line)" disclosure with content like "I need to r" before the rest of the word landed in the next chunk. - "Let me fix X." style stall endings now get flagged as stalls. When a long refactor ends with the model saying it will do something — "Let me fix all three project cards at once." — but the turn ends without the corresponding tool call, the response now picks up the same "[Bandit announced this action but did not emit the tool call]" suffix you already see for colon-ending ("Let me revert it:") narrate stalls. ## 1.7.364

- **Fixes a false-positive stall on casual greetings.** Asking "how are you this evening?" (or any other small-talk question) used to send Bandit into the no-tool-call recovery ladder, retrying the same conversational answer five times before bailing out with "I got stuck — emitted 5 responses in a row without successfully invoking a tool." The announce-intent detector's progress-narration branch was matching ANY "I'm [word]ing" pattern, which means `I'm doing well` looked exactly like `I'm porting the runtime` to the heuristic. Fix: the gerund branch now requires a syntactic complement after the -ing verb (a preposition / article / object pronoun like `on`, `with`, `the`, `it`), so "I'm doing well" / "I'm feeling fine" / "I'm going home" no longer trip the detector. Real progress narration ("I'm currently porting the agent runtime") still does. Three regression-pinning contract tests added.

## 1.7.363

- **Long refactor turns survive a second reasoning stall.** The recovery ladder (`empty_retry` → `thinking_off_recovery` → `prefill_recovery`) was treating prefill as a once-per-turn budget. On long turns that recovered from one stall and then hit another later — captured 2026-06-02 on a 26-iteration site build that succeeded at iter 25 and then stalled again at iter 26 — the second stretch fell through to the terminal "Bandit stalled" message even though every other detector still had budget. Now the prefill budget resets whenever a real tool call lands, so each independent stretch of failures gets the full ladder. The per-stretch hard cap (5 no-tool-call responses) still bounds the worst case.
- **Recovery-ladder events are visible in the UI.** `tool_loop:prefill_recovery` now surfaces as a status pill ("Prefilling tool envelope to break reasoning stall…") and a turn-log entry in the extension; the CLI also gains turn-log + dim-info-line renderings for `empty_retry`, `thinking_off_recovery`, and `prefill_recovery`. Previously the ladder was firing silently — users saw a multi-second pause with no indication that anything was happening between iterations.

## 1.7.301

- **Secret redaction now catches camelCase JSON config values and `Authorization: Bearer` headers.** The redactor's last-resort pattern only matched SHOUTY_CASE env-style names (`GITHUB_TOKEN=...`, `*_KEY=...`), so values inside JSON config files — including the `apiKey` field of `~/.bandit/config.json` — could surface in tool output, terminal scrollback, and session logs without ever being redacted. Both preserve the field name / header / scheme so the model still knows what kind of credential was hidden — only the value is replaced with a typed `<REDACTED:...>` placeholder.
- **`web_fetch` refuses private, loopback, and link-local addresses unless explicitly enabled.** Previously the tool would GET any URL the model produced, including `http://169.254.169.254/latest/meta-data/` (cloud metadata), `http://localhost:6443` (local kube API), or any RFC1918 internal endpoint. The tool now resolves the hostname before fetch and blocks IPv4 `10.*`, `172.16-31.*`, `192.168.*`, `127.*`, `169.254.*`, `100.64-127.*` (CGNAT), `0.0.0.0/8`, plus IPv6 `::1`, `::`, `::ffff:<private-v4>`, `fc00::/7` (ULA), `fe80::/10` (link-local). Set `BANDIT_ALLOW_PRIVATE_WEB_FETCH=1` to opt back in when you genuinely want the agent reaching an internal endpoint.
- **Custom skill `command:` substitution preserves each parameter as a single argv entry.** Templates used to substitute `{{param}}` and THEN split on whitespace, so a parameter value containing spaces or shell metacharacters fragmented into multiple argv slots (`git {{op}}` with `op="log; touch /tmp/x"` became `['git', 'log;', 'touch', '/tmp/x']`). `runCommand` doesn't shell out so no shell interpretation occurred — but the extra tokens still reached the binary as separate args, which is a privilege-escalation vector for commands taking `--exec` / `--upload-pack` style flags. Templates now split first, then substitute per-token, so each parameter value stays exactly one argv element regardless of contents.
- **`PreToolUse` hook output and the `/memory` preview now pass through the secret redactor.** Blocking-hook stdout/stderr was shown in plaintext and written to `.bandit/turns/*.jsonl` without redaction — if a validation hook echoed a credential, it leaked. `/memory` displayed BANDIT.md / CLAUDE.md content directly, so a user who pasted a config example into their memory file would see the secret echoed on every invocation. ## 1.7.298

- **`apply_patch` applies hunks independently with whitespace-tolerant fallback — one bad hunk no longer poisons a multi-file patch.** Right-way fix #3. Prior behavior: a 5-file unified diff with 12 hunks would abort the ENTIRE patch on the first hunk whose context line had drifted (e.g. from a previous edit in the same session shifting whitespace by one character). Model then had to regenerate the whole patch. New behavior: each hunk attempts independently and is reported per-hunk; the file is written with whatever applied, and partial success returns `isError:false` so the model can rebuild just the failing hunks instead of starting over. The individual stall detectors (`empty_retry`, `narratedButNoAction`, `thinking_off_recovery`, `parse_retry`, `prose_loop`) each had their own caps, but they chained. `thinking_off_recovery` resets the consecutive-empty counter to zero, and the model could move between failure modes faster than any one detector could give up. ## 1.7.258

- **CLI: spinner now actually pauses while you're typing a follow-up.** The `pauseFor()` mechanism existed but was half-wired — `render()` never checked the pause window, so the 80ms repaint timer kept firing through the pause and wiped any character readline had echoed to the screen. ## 1.7.137

## 1.7.136

- **Auto-context no longer fishes the workspace for keyword-matching source files.** The old behavior scored every file in the workspace index against prompt tokens and pre-attached up to 3 matches — e.g. "update the api controllers" auto-attached `FileController.cs` + `HealthController.cs` before the model saw your prompt, eating 15+ KB per turn. Stripped the scoring loop entirely. When the toggle is on, auto-context now attaches only: (a) the **active editor file** if you have one open, and (b) any **explicit file paths** you type in your prompt (e.g. `src/foo.ts`). Project memory files (BANDIT.md / CLAUDE.md) still inject on every turn via their own path.
- **Default changed to OFF.** Toggle in the composer or set `banditStealth.autoContextEnabled` to opt in.

## 1.7.33

- **Auto-context toggle is now persisted.** New `banditStealth.autoContextEnabled` setting (default `true`, scoped globally). Flipping the toggle in the composer writes the config and survives webview reloads, window restarts, and new workspaces — no more silent re-enable.
- Marketing polish on the README: tighter "independent shop" framing, dropped infrastructure-specific details from the model table, and relocated voice out of the headline bullets since it requires the cloud provider.

## 1.7.32

- **CLI @-picker no longer stacks ghost copies.** Arrow-key navigation used to leave a fresh copy of the picker list under the previous one each keystroke (save/restore cursor anchoring broke once the list scrolled). Switched to explicit line-count tracking — redraw walks up N lines and erases each.
- **IDE @-picker now finds folders properly.** Prior behavior derived folders only from ancestors of matching files, so typing `@src` couldn't find `src/` unless a file inside it also contained "src" in its name. Added a depth-capped directory enumeration (≤4 levels, ≤400 dirs) that matches folder names independently. Empty-query `@` now also surfaces top-level workspace folders.
